## 🐛 Reporting a Vulnerability

### How to Report

If you discover a security vulnerability, please report it by:

1. **Email**: (Añadir email si quieres)
2. **GitHub Security**: Use GitHub's [private vulnerability reporting](https://github.com/pabloalgo/pabloib/security/advisories/new)

**Please do NOT create a public issue for security vulnerabilities.**

### What to Include

- Type of vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Time

- **Initial response**: Within 48 hours
- **Status update**: Within 7 days
- **Resolution**: Depends on severity

## 🔐 Security Measures

This project implements:

- ✅ HTTPS enforcement
- ✅ Content Security Policy headers (recommended)
- ✅ Dependency scanning
- ✅ Regular security updates
- ✅ No hardcoded secrets
- ✅ Secure dependencies

## 🛡️ Best Practices

### For Contributors

- Never commit sensitive data (API keys, passwords)
- Use environment variables for secrets
- Keep dependencies up to date
- Follow secure coding practices

### For Users

- Always use HTTPS
- Keep your Hugo version updated
- Review third-party themes before use
- Regularly update dependencies
